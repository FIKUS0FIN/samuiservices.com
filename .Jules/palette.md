## 2026-07-10 - Form Labels Accessibility
**Learning:** React elements without explicit IDs can't rely on generic strings to maintain correct linking for `htmlFor` - using `React.useId()` allows standard components (like `<Input>`) to generate unique IDs and guarantee pairing between label and input for screen readers. Explicit `id` must be tied directly to `<select>` and `<textarea>` nodes inside custom forms.
**Action:** Next time, always check if `<label>` elements missing an `htmlFor` have a corresponding `id` on their target form fields or implement a reliable ID-generation approach in component props to fix it.
